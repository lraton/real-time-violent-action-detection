# LSTM Training Script

This script (`src/train_lstm.py`) trains a Bidirectional LSTM neural network to classify video sequences as "Violent" or "Non-Violent" using pose keypoints.

## Code Overview

### Dependencies
-   `tensorflow.keras`: For building and training the LSTM model.
-   `sklearn`: For `train_test_split` and `compute_class_weight`.
-   `numpy`: Array manipulation.
-   `glob`, `os`: File system operations.

### Configuration
Key hyperparameters defined at the start of the script:
-   `MAX_FRAMES = 150`: Input sequences are padded or truncated to this length.
-   `BATCH_SIZE = 32`
-   `EPOCHS = 100`
-   `MASK_VALUE = -999.0`: Special value used for padding sequences.

### Main Workflow

1.  **Data Loading**:
    -   Scans `../datasets/lstm_dataset/` for `.npz` files.
    -   Loads feature arrays (`data`) and labels (`label`) into lists `X` and `y`.

2.  **Preprocessing (`preprocess_data`)**:
    -   **Padding**: Uses Keras `pad_sequences` to ensure all inputs have length `150`.
        -   Padding/Truncating direction: `post` (at the end).
        -   Value: `-999.0` (ignored by the model due to masking).
    -   **Reshape**: Flattens the feature dimensions if necessary to match `(samples, 150, features)`.

3.  **Model Architecture (`create_and_train_model`)**:
    The model uses a Sequential architecture designed for sequential data:
    -   **Masking Layer**: Ignores time steps with value `-999.0`.
    -   **Bidirectional LSTM (64 units)**: Captures temporal context in both forward and backward directions.
    -   **Dropout (0.3)**: Regularization to prevent overfitting.
    -   **Bidirectional LSTM (32 units)**: Second recurrent layer.
    -   **Dropout (0.3)**
    -   **BatchNormalization**: Stabilizes learning.
    -   **Dense (32, ReLU)**: Fully connected layer.
    -   **Dropout (0.2)**
    -   **Dense (1, Sigmoid)**: Output layer for binary classification (0 to 1).

4.  **Training Process**:
    -   **Split**: 80% Training, 20% Validation (Stratified to maintain class balance).
    -   **Class Weights**: Automatically computed to balance the loss function against uneven class distributions (Violent vs Non-Violent).
    -   **Callbacks**:
        -   `ModelCheckpoint`: Saves the best model to `../models/lstm_violence_detector_tmp.keras` monitoring `val_loss`.
        -   `EarlyStopping`: Stops training if `val_loss` doesn't improve for 10 epochs.
        -   `ReduceLROnPlateau`: Halves the learning rate if validation loss plateaus for 3 epochs.

## Outputs
-   A trained Keras model saved at: `../models/lstm_violence_detector_tmp.keras`.
-   Console output showing model summary, training progress, and class weights.

## Usage
Run the script directly from the terminal:
```bash
python src/train_lstm.py
```
*Prerequisite: The dataset folder `../datasets/lstm_dataset/` must contain `.npz` files generated by the preprocessing script.*
